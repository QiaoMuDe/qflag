# 智能纠错功能影响分析文档

## 功能概述

智能纠错功能在解析命令时，当用户输入错误的子命令或标志时，自动提供相似的建议，类似于 Git 的 "did you mean" 功能。

## 实现组件

1. **错误类型** (`internal/types/error.go`)
   - `UnknownSubcommandError`: 未知子命令错误
   - `UnknownFlagError`: 未知标志错误

2. **建议查找器** (`internal/parser/suggestion.go`)
   - `SuggestionFinder`: 封装查找逻辑
   - `checkUnknownFlags()`: 预扫描未知标志

3. **解析器集成** (`internal/parser/parser.go`)
   - `Parse()`: 集成子命令纠错
   - `ParseAndRoute()`: 集成子命令纠错
   - `ParseOnly()`: 集成标志纠错

## 判断逻辑

### 子命令纠错

```
条件：
1. 命令有子命令
2. 第一个参数不以 "-" 开头（避免将标志误判为子命令）

结果：如果不匹配任何子命令，返回带建议的错误
```

### 标志纠错

```
条件：
1. 参数以 "-" 开头
2. 不在已注册标志列表中

边界处理：
- 遇到 "--" 停止扫描（后面视为位置参数）
- 遇到子命令名停止扫描（后续标志由子命令处理）

结果：返回带建议的错误
```

## 潜在影响分析

### 1. 向后兼容性

**风险等级**: 中

**影响**: 之前输入错误子命令或标志时，行为可能不同：
- 子命令：之前忽略错误参数，现在报错
- 标志：之前由标准库处理错误，现在预检查报错

**缓解措施**: 
- 错误信息更友好，帮助用户快速定位问题
- 建议列表限制为3个，避免信息过载

### 2. 性能影响

**风险等级**: 低

**影响**: 
- 每次解析时额外扫描参数列表（O(n)）
- 使用 fuzzy 匹配算法（O(m*n)，m为候选数，通常很小）

**缓解措施**:
- 扫描提前终止（遇到子命令或 "--"）
- 候选列表通常很小（子命令和标志数量有限）

### 3. 边界情况

#### 3.1 禁用标志解析

**场景**: 命令设置了 `SetDisableFlagParsing(true)`

**行为**:
- 标志纠错：不执行（ParseOnly 直接返回）
- 子命令纠错：正常执行，但参数以 "-" 开头的视为普通参数

**潜在问题**: 无

#### 3.2 嵌套子命令

**场景**: `parent child grandchild --flag`

**行为**:
- 遇到子命令名时停止标志扫描
- 子命令负责处理自己的标志

**潜在问题**: 无

#### 3.3 隐藏命令

**场景**: 内置补全命令 `__complete`

**行为**:
- 隐藏命令不会出现在建议列表中

**潜在问题**: 无

#### 3.4 长短名称

**场景**: 子命令或标志有长短两个名称

**行为**:
- 建议列表包含长短两个名称
- 用户输错时可能匹配到长名或短名

**潜在问题**: 建议列表可能同时显示长短名称（如 `config` 和 `cfg`）

### 4. 错误信息格式

**风险等级**: 低

**影响**: 错误信息格式改变，可能影响依赖错误信息的自动化工具

**缓解措施**: 
- 错误信息包含明确的错误类型前缀
- 建议列表格式统一

### 5. 模糊匹配准确性

**风险等级**: 中

**潜在问题**:
- 用户输入与多个候选相似时，建议顺序可能不符合预期
- 极端短输入（如单个字符）可能匹配不准确

**缓解措施**:
- 限制建议数量为3个
- 使用 fuzzy.CompletePrefix 算法（前缀优先）

## 测试覆盖建议

1. **基本功能测试**
   - 子命令纠错
   - 标志纠错
   - 长短名称匹配

2. **边界情况测试**
   - 禁用标志解析
   - 嵌套子命令
   - "--" 终止符
   - 隐藏命令过滤

3. **错误信息测试**
   - 格式正确性
   - 建议准确性

## 总结

智能纠错功能整体风险可控，主要影响是：
1. 错误行为从"静默忽略"变为"主动报错"（更友好）
2. 性能影响极小（额外扫描一次参数列表）
3. 边界情况已处理（禁用解析、嵌套命令、隐藏命令等）

建议在实际使用场景中充分测试，特别是复杂命令结构的应用。
